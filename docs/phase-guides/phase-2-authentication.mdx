## Status: ✅ Complete

## Highlights

<CardGroup cols={2}>
  <Card title="JWT Auth" icon="key">
    `POST /api/auth/login` issues a 7-day JWT. The frontend stores it in `localStorage` and attaches it to every Axios request.
  </Card>
  <Card title="bcrypt Hashing" icon="lock">
    All passwords are hashed with bcryptjs. Plain-text passwords never touch the database.
  </Card>
  <Card title="Role-Based Access" icon="user-shield">
    Three roles enforced in middleware AND in `ProtectedRoute` on the frontend.
  </Card>
  <Card title="Audit Logging" icon="clipboard-list">
    `audit_logs` table records login attempts, password changes, and role updates.
  </Card>
</CardGroup>

## New Endpoints

| Method | Path | Purpose |
|--------|------|---------|
| POST | `/api/auth/login` | Authenticate and get JWT |
| POST | `/api/auth/logout` | Invalidate session |
| GET | `/api/auth/profile` | Get current user |
| POST | `/api/auth/change-password` | Update password |

See [Authentication API](/api-reference/auth) for the full reference.

## New Tables

- `user_sessions` — tracks active JWTs server-side
- `roles` — role definitions
- `audit_logs` — security event log

## Frontend Pieces

- `AuthContext` (global auth state)
- `useAuth()` custom hook
- `Login.jsx` with form validation
- `ProtectedRoute` component
- Role-based nav in `Layout.jsx`
